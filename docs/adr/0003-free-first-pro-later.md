# ADR-0003: Launch Free First, Add Pro Later

## Status

Accepted.

## Context

The app is the first project for the developer. Ads conflict with a distraction-reduction product, and subscriptions create an obligation to provide ongoing paid value.

## Decision

Launch V1 free and without ads. After validating usage, add a one-time Pro unlock for advanced settings.

## Consequences

Positive:

- Easier first launch.
- Cleaner user trust story.
- No ad SDK or billing complexity in V1.
- Monetization can be based on real user feedback.

Tradeoffs:

- No early revenue.
- Pro feature boundaries need careful design later.
